# Ingeniería del Index: Grids y Sticky Columns

El catálogo de Tyac no es un simple listado vertical. Diseñamos un sistema de cuadrícula inteligente que aprovecha el espacio en pantallas grandes y se adapta perfectamente a móviles, usando una técnica de **columnas pegajosas (Sticky Columns)**.

## 1. El diseño de dos columnas

En la sección de Cursos y Apuntes del inicio, dividimos la pantalla en dos partes:
1.  **Info-Sticky (Izquierda)**: Contiene el título de la sección, la descripción y los filtros.
2.  **Contenido-Scroll (Derecha)**: La lista de tarjetas que el usuario puede recorrer.

### ¿Por qué esta estructura?
Permite que el usuario siempre sepa en qué sección se encuentra (Cursos vs Apuntes) y tenga los filtros a mano mientras explora las tarjetas.

## 2. Implementación con CSS Grid

Usamos un grid fraccional para dar prioridad al contenido:

```css
.seccion-grid {
    display: grid;
    grid-template-columns: 1fr 2.5fr; /* 1 parte para info, 2.5 para tarjetas */
    gap: 4rem;
    align-items: start;
}

.info-sticky {
    position: sticky;
    top: 100px; /* Se queda fijo cuando llegas a su altura */
}
```

## 3. Inversión Visual

Para romper la monotonía, en la sección de **Apuntes** invertimos las columnas: el contenido está a la izquierda y la info sticky a la derecha. Esto se logra fácilmente con una clase `.invertida`:

```css
.seccion-grid.invertida {
    grid-template-columns: 2.5fr 1fr;
}
```

## 4. Responsividad Automática

En móviles (menos de 900px), el grid se transforma en una sola columna mediante una **Media Query**, y la información deja de ser "sticky" para fluir naturalmente hacia abajo:

```css
@media (max-width: 900px) {
    .seccion-grid { grid-template-columns: 1fr; }
    .info-sticky { position: static; }
}
```

> [!IMPORTANT]
> **Punto Clave**: Esta técnica de diseño mejora drásticamente la UX al reducir el scroll innecesario y mantener el contexto de navegación siempre visible.
