# Estructura Base: El Layout Maestro

En Tyac, el archivo `src/layouts/head/00-Layout.astro` es el orquestador principal. Es el "molde" que define la estructura global de toda la plataforma. Sin este archivo, Tyac sería simplemente una colección de páginas desordenadas.

## 1. El concepto de Envoltura Global

Imagina el layout como un marco de cuadro. El contenido de cada página (como este texto) es la pintura, y el layout es el marco que contiene el cristal, el borde y el soporte.

## 2. Los 3 Pilares del Layout Base

El archivo `00-Layout.astro` se encarga de tres cosas fundamentales:

### A. Estructura HTML5 Semántica
Define el `<html>`, `<body>` y la ubicación del `<main>`. Usa el elemento `<slot />` para decidir exactamente dónde debe aparecer el contenido de cada página.

### B. Elementos Persistentes
Aquí es donde inyectamos el **Navbar** y el **Footer**. Como están definidos en el Layout Maestro, aparecen en todas las páginas automáticamente sin tener que importarlos 50 veces.

### C. Sistema de Transiciones
En este archivo es donde activamos el `<ClientRouter />`. Esto permite que cuando navegues entre lecciones, la página no parpadee en blanco, sino que los elementos se deslicen suavemente.

## 3. Ejemplo de Código en Tyac

```astro
---
// 00-Layout.astro
import Metadata from "./01-Metadata.astro";
import Navbar from "./02-Navbar.astro";
import Footer from "../../components/Footer.astro";
import { ClientRouter } from "astro:transitions";
---
<html lang="es">
    <head>
        <Metadata title={titulo} />
        <ClientRouter />
    </head>
    <body>
        <Navbar />
        <main>
            <slot /> <!-- Aquí estalla la magia -->
        </main>
        <Footer />
    </body>
</html>
```

> [!TIP]
> **Consistencia Visual**: Gracias a este layout, si añadimos una variable de CSS en `body`, afectará a TODO Tyac al instante.
