# Variables Semánticas vs Colores Sueltos

En la guía anterior vimos los Design Tokens. Ahora vamos a profundizar en cómo los usamos mediante **Variables Semánticas**. En Tyac, no pintamos un botón de "púrpura"; lo pintamos con el color de "Acento". 

## 1. El concepto Semántico

Una variable semántica describe la **función** del color, no su apariencia. Por ejemplo:

- **Malo (No semántico)**: `--color-morado` (¿Qué pasa si queremos que ese botón sea azul mañana?).
- **Bueno (Semántico)**: `--color-brand-primary` (Representa la marca, sin importar qué color real sea).

## 2. Implementación de Capas en Tyac

En Tyac usamos un sistema de capas:
1.  **Capa de Tokens**: Definimos los colores base (`--dracula-purple`).
2.  **Capa Semántica**: Asignamos esos tokens a funciones específicas.

```css
:root {
  /* Tokens */
  --t-purple: #bd93f9;
  
  /* Semántica */
  --clr-accent: var(--t-purple);
  --btn-bg-primary: var(--clr-accent);
  --border-active: var(--clr-accent);
}
```

## 3. Por qué es vital para el Mantenimiento?

Si decidimos que Tyac necesita una "Edición Especial" con colores de Navidad, solo tenemos que cambiar la Capa Semántica. Todos los botones, bordes y links que usan `--clr-accent` cambiarán al unísono, asegurando que no se nos escape ningún rincón de la web.

## 4. Uso en Componentes Astro

Al usar variables semánticas en nuestros componentes (como `Buscador.astro`), el código se vuelve mucho más legible. Cualquier desarrollador que lea el CSS entiende que ese color es el de acento, sin tener que memorizar códigos hexadecimales.

> [!IMPORTANT]
> **Norma Tyac**: Queda prohibido usar colores hexadecimales (`#...`) dentro de los componentes. Siempre se debe invocar una variable semántica para garantizar la integridad visual del proyecto.
