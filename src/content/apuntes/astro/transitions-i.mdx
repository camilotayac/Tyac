# Transiciones de Vista I: ClientRouter

¿Te has fijado en que al navegar por Tyac la página no "muere" ni se queda en blanco por un segundo? Eso es gracias a la API de **View Transitions** y al componente `<ClientRouter />` de Astro. Hemos transformado un sitio estático en una "App" fluida donde cada cambio de página es cinematográfico.

## 1. Qué es el ClientRouter?

Por defecto, los navegadores eliminan todo lo que hay en pantalla cuando haces clic en un enlace y empiezan a dibujar la nueva página desde cero. El `<ClientRouter />` intercepta esos clics y realiza el cambio bajo el capó, permitiendo que el navegador mantenga elementos en pantalla durante la transición.

## 2. Implementación Global

En Tyac, activamos esta magia en el `00-Layout.astro` para que afecte a todas las páginas:

```astro
---
import { ClientRouter } from 'astro:transitions';
---
<head>
    <ClientRouter />
</head>
```

## 3. Animaciones Automáticas

Solo con añadir esa línea, Tyac ya tiene animaciones. Pero hemos ido más allá personalizando cómo se comportan:
- **Fade**: El contenido viejo se desvanece mientras el nuevo aparece.
- **Slide**: En el sidebar, el contenido se desliza lateralmente, dando la sensación de que estás explorando un panel continuo.

## 4. El fin del "Flash" Blanco

Gracias a esta tecnología, Tyac elimina el molesto parpadeo blanco al navegar. Esto es vital para una plataforma e-learning, ya que mantiene al estudiante en un estado de concentración absoluta (flow), sin interrupciones visuales bruscas.

> [!IMPORTANT]
> **Compatibilidad**: Aunque las View Transitions son modernas, Astro se encarga de que, si un usuario usa un navegador antiguo, Tyac siga funcionando perfectamente de la forma tradicional, asegurando que nadie se quede fuera.
