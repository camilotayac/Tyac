# UI Atómica: Buscadores y Tarjetas

En Astro, un componente es un archivo `.astro` que combina HTML, CSS y JavaScript. En Tyac, hemos diseñado componentes que no solo se ven bien, sino que son inteligentes.

## Anatomía de un componente Tyac

Tomemos como ejemplo nuestro buscador híbrido. Se divide en tres secciones:

### 1. El Script de Servidor (Component Script)
Es el bloque entre `---`. Aquí definimos qué "Props" recibe el componente. Por ejemplo, el `mode` ("hero" o "header"):

```astro
---
interface Props {
    mode?: "hero" | "header";
}
const { mode = "hero" } = Astro.props;
---
```

### 2. El Esqueleto (Component Template)
Es el HTML puro. Usamos clases dinámicas basadas en las props para cambiar el diseño sin duplicar código:

```astro
<div class={`buscador-global-container ${mode}`} data-buscador>
    <input type="text" class="buscador-input" placeholder="..." />
</div>
```

### 3. El Estilo (Scoped CSS)
En Tyac usamos CSS local. Esto significa que los estilos del buscador no "se filtran" a otros elementos de la página, evitando errores visuales:

```astro
<style>
    .buscador-inner:hover {
        transform: translateY(-4px) scale(1.02);
        border-color: var(--dracula-purple);
    }
</style>
```

## Componentes con vida propia

Algunos de nuestros componentes necesitan JavaScript en el navegador (como el buscador). Astro nos permite incluir etiquetas `<script>` que se autoejecutan solo cuando el componente aparece en pantalla.

> [!TIP]
> **Dato Tyac**: Usamos el atributo `is:inline` o etiquetas `script` estándar para manejar la interactividad del modal de búsqueda, asegurando que el filtrado sea instantáneo y fluido.
